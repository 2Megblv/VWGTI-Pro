# 02-01-PLAN: Phase 1 Refactoring into Modular Headers

## Objective

Refactor the Phase 1 monolithic EA code (single .mq5 file) into three reusable header files (VolumeProfile.mqh, RiskManager.mqh, Utils.mqh) while preserving all Phase 1 functionality. This modularization is required before Phase 2 signal detection and execution logic can be cleanly integrated.

**Purpose:** Separate concerns (profile calculation vs risk management vs execution) for maintainability and testability.

**Output:** Three .mqh header files compiled and verified; main EA file refactored to #include headers; all Phase 1 behavior unchanged.

---

## Success Criteria (Goal-Backward Verification)

- [ ] VolumeProfile.mqh compiles without errors and exports CalculateCurrentVolumeProfile(), CalculateValueArea(), IdentifyVolumeNodes()
- [ ] RiskManager.mqh compiles without errors and exports CalculateLotSize(), CheckDailyHardStop(), CheckProfitCap(), CheckFridayClose()
- [ ] Main EA file (VolumeProfile_EA_v1.0.mq5) #includes both headers and compiles successfully
- [ ] OnTick() flow unchanged: recalculates profile, enforces daily limits, returns correct profile state
- [ ] Phase 1 unit tests still pass (profile accuracy ±0.1%, daily limit enforcement verified)
- [ ] All struct definitions and constants migrated to appropriate header files

## Key Dependencies

- Phase 1 source code (src/VolumeProfile_EA_v1.0.mq5) with embedded unit tests
- MT5 Build 4000+ compilation environment
- No external dependencies introduced (all refactoring is internal code organization)
